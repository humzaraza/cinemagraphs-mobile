Curation locked: 57 unique films, 32 era picks + 36 genre picks (with 11 cross-block duplicates per relaxed dedup rule). Documentary genre dropped after curation iteration.
Locked decisions:
• Banner output is BACKDROP type. No schema work. Existing bannerType=BACKDROP with bannerValue={filmId, backdropPath} JSON.
• Fallback hierarchy: complete onboarding + Screen 3 picks → BACKDROP from chosen film. Skip Screen 3 only OR no usable backdrop → GRADIENT preset mapped from era/genre taste. Skip everything → default GRADIENT (midnight).
• Backdrop selection algorithm: top film by IMDb rating among Screen 3 picks that ALSO has a quality TMDB backdrop (width ≥1280, no language-tagged title cards). Same filters as /api/films/[id]/backdrops route from PR 1c.
• Screen 1: 8 era blocks (1920s-30s, 1940s-50s, 1960s-70s, 1980s, 1990s, 2000s, 2010s, 2020s). Max 4 selections. Cap overflow = no-op with shake animation.
• Screen 2: 9 genre blocks (Drama, Horror, Comedy, Sci-fi, Thriller, Romance, Action, Animation, Crime). Max 5 selections. Cap overflow = no-op with shake.
• Screen 3: 12 individual film posters with titles + years. Top 12 by IMDb votes among matching era+genre intersection. Excludes posters already shown in mosaic blocks. Adjacent-decade fallback if <12 matches. No upper cap on user selections.
• Microcopy: Screen 1 "Pick the eras you keep returning to". Screen 2 "Pick your genres". Screen 3 "Pick the films you love".
• Block selection: tap one mosaic block as a unit; gold border + scale 1.05 + checkmark badge.
• Cross-screen accumulation: posters from selected blocks compress into a horizontal cascade strip at the top of the next screen. Live-updates on deselect. Right-edge fade truncate (no horizontal scroll).
• Continue gating: Continue button hidden until first selection on each screen. Skip top-right is the explicit zero-selection path.
• Dissolution timing: 2 to 2.5 seconds total. Magnet-style convergence (posters stream inward in two arcs from screen edges with deceleration easing). Banner emerges from center with subtle glow.
• Reveal: bio row hidden when empty. No source attribution pills.
• Done CTA destination: router.replace('/(tabs)/explore') matching current onboarding.
• Catalog source for mosaic posters: catalog-only (closed; verified via top-N-by-IMDb-votes catalog query).

Open questions and parked followups

Items deliberately deferred or that need a decision before code starts.

1. CLOSED: Catalog source for mosaic posters. Catalog-only, confirmed via top-N-by-IMDb-votes catalog query during curation. Documentary block dropped after iteration revealed insufficient cohesive 4-poster sets.

2. CLOSED: Max-selection overflow. No-op with shake animation. Implementation: Reanimated withSequence(withTiming(4), withTiming(-4), withTiming(0)), ~200ms total. Tactile feedback that the cap exists, no auto-deselect.

3. CLOSED: Right-edge fade on accumulation strip. No horizontal scroll (would compete with grid scroll gesture). CSS mask-image: linear-gradient(to right, black 80%, transparent 100%). Decorative, no interaction.

4. CLOSED: Hide bio row on reveal if empty. No "Bio coming soon" placeholder. Users add bio later from Settings.

5. Accumulation strip dedup behavior. 11 films appear in both an era and a genre block per the relaxed dedup rule from curation. When user selects an era AND a genre that share a film, does the strip show that film once or twice? Mockup shows duplicates for visual fidelity to the selection set. Real impl could either: (a) match mockup and show duplicates, or (b) dedup at strip-render time. Lean toward (b) for cleaner visual but not blocking.

6. Skip Screen 1 / Skip Screen 2 candidate pools. If user skips Screen 1, what does Screen 2 show? All 9 genre mosaics with default posters (already curated). If user skips Screen 1 AND Screen 2, what does Screen 3 show? Top 12 popular films overall. Reasonable defaults; confirm before build.

7. Backdrop algorithm tie-breaking. If top-rated Screen 3 pick has no usable backdrop, fall to second-rated. If none of the picks has a usable backdrop, fall through silently to GRADIENT mapped path. Confirmed.

8. Already-shown poster exclusion. Confirmed: Screen 3 algorithm excludes the 57 unique films shown in mosaic blocks. Adjacent-decade fallback if <12 candidates remain. SQL count check on catalog before build to validate this won't deadlock for narrow selection combos (e.g., user picks only 1920s-30s + Horror).

9. Reveal screen avatar. Mockup shows "H" letter. OAuth avatar from Google/Apple sign-in should be used if present at this point in the flow; letter fallback only if not. Build-time concern.

10. Existing app/onboarding.tsx delete. The Prompt 10 carousel currently lives at this path. PR 2 replaces it. About.tsx covers the same content for depth-seekers. Delete in the same PR as new onboarding ships.
